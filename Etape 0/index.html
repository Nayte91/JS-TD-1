Bienvenue dans ce TD !

L'objectif premier du javascript est de suppléer le HTML et le CSS dans les pages web, afin d'apporter des interactions et effets qui méritent de la logique de code.

On peut le trouver impliqué dans les animations visuelles, la récupération d'informations depuis d'autres sites (API), l'afficahge dynamique de contenu, ... Par exemple, quand on a une carte Google map sur un site, cela coche toutes les cases du JavaScript.

Et l'un des cas classiques du genre est le menu déroulant : faire qu'un contenu se mette à jour lorsque l'on choisi un item dans une liste.

Personne n'a envie de sélectionner une entrée, puis de cliquer sur "choisir", et enfin d'attendre que la nouvelle page se charge pour voir le résultat. Encore plus quand on doit les enchainer : un formulaire d'adresse postale avec "pays" --> charger --> "ville" --> charger --> "rue" --> charger... Serait insupportable si on devait valider à chaque fois.

Donc nous allons jouer avec des data qu'on aime bien, pour les afficher proprement... :)

Ci-dessus est un menu déroulant vide, qui en HTML se fait avec une balise select. Les options à l'intérieur sont d'autres balises option.

Le but du jeu va donc être de remplir son contenu dynamiquement, et de le faire afficher quelques infos en fonction du choix. Ensuite, on verra où ça nous mène !

Pour faire cela, on va avoir besoin de manipuler le navigateur internet et le contenu de notre page. Et pour ça, on a quelques concepts javascript liés au HTML à manipuler. Je vais tenter de les expliquer au mieux.

Comme souvent en web, la logique de code pure est mêlée à l'affichage, donc il est normal de patauger au milieu des fonctions inconnues pour au final écrire peu de logique. Ici, on aura quand même besoin de manipuler du array, avec du foreach, du map et du filter, ainsi que de l'objet, et de mixer tout cela autour d'une structure imbriquée !
